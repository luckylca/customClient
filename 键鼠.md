# 自定义客户端通信协议规范 (v1.0)

本规范定义了自定义客户端与机器人之间的控制指令交互（基于 MQTT/Protobuf）及图传数据传输（基于 UDP/HEVC）的标准。

## 1. 网络配置 (Network Configuration)

| 配置项 | 属性 | 值 | 备注 |
| :--- | :--- | :--- | :--- |
| **服务器 IP** | 固定 IP | `192.168.12.1` | 机器人端/车载端 |
| **客户端 IP** | 固定 IP | `192.168.12.2` | 开发设备/自定义控制器 |
| **MQTT 端口** | Port | `3333` | 用于指令传输 |
| **UDP 端口** | Port | `3334` | 用于图传数据接收 |

---

## 2. 指令交互协议 (MQTT + Protobuf)

### 2.1 传输规范
* **序列化格式**：Protobuf v3 (支持 v3.15+)
* **传输协议**：MQTT (直接传输二进制流)
* **Topic 命名**：与指令名（Message Name）一致
* **QoS 等级**：最高支持 QoS 1
* **发布频率**：建议 **75Hz** (约 13.3ms 间隔)

### 2.2 开发流程
1.  **定义接口**：编写 `.proto` 文件。
2.  **生成代码**：使用 `protoc` 编译器生成对应语言（C++/Python/Java等）的类。
3.  **封包发送**：发布者将对象序列化为二进制流，通过 MQTT 发送至对应 Topic。
4.  **解包接收**：订阅者接收二进制流，使用反序列化接口还原对象。

## 3. 详细协议定义

### 3.1 KeyboardMouseControl (键鼠控制指令)
**用途**：实时传输操作手的键盘与鼠标输入，用于远程驾驶或自动化控制。

#### Protobuf 结构定义
```protobuf
syntax = "proto3";

message KeyboardMouseControl {
    int32 mouse_x = 1;          // 鼠标 X 轴移动速度（负值向左）
    int32 mouse_y = 2;          // 鼠标 Y 轴移动速度（负值向下）
    int32 mouse_z = 3;          // 鼠标滚轮移动速度（负值向后）
    bool left_button_down = 4;  // 左键状态
    bool right_button_down = 5; // 右键状态
    bool mid_button_down = 7;   // 中键状态
    uint32 keyboard_value = 6;  // 键盘按键位掩码 (Bit0 - Bit15)
}
```

#### 键盘位掩码 (keyboard_value) 映射表
`uint32` 的每个 bit 代表一个按键的状态（1 为按下，0 为抬起）：

| Bit | 按键 | Bit | 按键 | Bit | 按键 | Bit | 按键 |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| **0** | W | **4** | Shift | **8** | R | **12** | X |
| **1** | S | **5** | Ctrl | **9** | F | **13** | C |
| **2** | A | **6** | Q | **10** | G | **14** | V |
| **3** | D | **7** | E | **11** | Z | **15** | B |

## 4. 图传链路协议 (UDP + HEVC)

客户端通过监听 **UDP 3334** 端口接收视频流。

### 4.1 码流格式
* **编码标准**：HEVC (H.265)
* **传输单元**：UDP 数据包

### 4.2 数据包结构 (Header: 8 Bytes)
每个 UDP 包的头部包含 8 字节的固定元数据，随后是原始 H.265 码流片段：

| 偏移量 | 长度 (Byte) | 字段名 | 说明 |
| :--- | :--- | :--- | :--- |
| `0x00` | 2 | **Frame ID** | 帧编号（单调递增，用于丢帧检测） |
| `0x02` | 2 | **Fragment No.** | 当前帧内的分片序号（组包使用） |
| `0x04` | 4 | **Total Size** | 当前帧的总字节数 |
| `0x08` | N | **Payload** | 原始 HEVC 码流数据 |

## 5. 开发建议与注意事项
1.  **clientID**：连接 MQTT 时，`MyMqttClient` 的 `clientID` 必须设置为机器人指定的 ID 编号，否则无法正确路由。
2.  **控制权切换**：比赛时需在官方选手端确认键鼠指令来源。若选择“自定义客户端”，官方端键鼠将失效。
3.  **丢包处理**：图传基于 UDP，开发者需根据 `Frame ID` 和 `Fragment No.` 自行处理分片组包逻辑，建议丢弃不完整的帧。
